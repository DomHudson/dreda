Dreda is a tool for exploring clustered data that has been reduced to three numerical dimensions. Often, the curse of dimensionality encourages one to use dimensionality reduction to turn high dimensional data into 3-dimensional data, a.k.a. data we can see!

Dreda takes a JSON object of objects as input - its set to accept the default 'index' ouput from a pandas DataFrame.to_json() function. The reasoning here is that pandas will allow you to make DataFrames based on SQL queries, CSVs, HDF, STATA, excel, as well as JSON from read_json() in a variety of ways.

Read more about Pandas IO here.

Prereqs

For Dreda to visualize your data, you'll need to meet a few requirements:

  1. valid JSON - maybe check with JSONLint
  2. 'index' style format output from a pandas DataFrame.to_json()
Format

Basically, the data is formatted as an object of objects. More specifically, we have three numerical values, and one categorical value that gets used as the coloring for the cluster. Within each object, we have key-value pairs. The key is the point-id (index value) and the value is, well, the value for that particular parent-object (a given dimension).

Scaling

Dreda is already set up to obtain the scale of the data and calculate the size of the grids and points accordingly, so don't worry about scale. However, it may be worth considering standardizing your data so it is displayed neatly across the origin (maybe with sklearn?).

It done broke

Lastly, I haven't written an elegant catch for badly formatted data. So right now if you try uploading something that doesn't work, feel free to check the console, see the error, and file an issue on github.
